## What is generated versus authored

| Path | Status | Regenerate with |
| --- | --- | --- |
| `assets/js/*.js` | Generated, committed | `npm run build` |
| `assets/images/*-<width>.*` | Generated, committed | `npm run images` |
| `assets/images/manifest.json` | Generated, committed | `npm run images` |
| `assets/vendor/three.min.js` | Vendored, committed | `npm pack three@0.128.0` |
| everything else | Authored | — |

Generated files are committed deliberately. Static hosts serve this repository
as-is with no build step, so the compiled output has to be present.

## Backend request flow

```
GET  /                     → index.html
GET  /api/health           → {"ok": true, "service": "ConstructX"}
GET  /api/*                → 404 JSON
GET  /assets/*             → file, gzipped if text, cached one year immutable
GET  /<anything else>      → file, or 404.html
GET  /<a directory>        → its index.html, or 404 (no directory listings)
POST /api/enquiries        → rate limit → parse → validate → spam check → store → email
POST /api/*                → 404 JSON
```

Response headers on every request: `Content-Security-Policy`,
`X-Content-Type-Options`, `Referrer-Policy`, `Permissions-Policy`,
`Cross-Origin-Opener-Policy`, `Cross-Origin-Resource-Policy`,
`X-Frame-Options`, and a path-appropriate `Cache-Control`.

## Adding a section to the page

1. Add the markup to `index.html` inside `<main>`. Give the `<section>` an `id`
   and point `aria-labelledby` at its heading.
2. Add `class="reveal"` to anything that should fade in on scroll. No JS needed —
   `initReveals()` picks it up from the class alone.
3. Add `class="sec-lazy"` only if the section is below the fold **and** its
   height is stable; it enables `content-visibility: auto`.
4. Add the anchor to `.nav-links` and the mobile menu if it belongs in
   navigation. `initSectionNav()` derives its scroll-spy targets from those
   links, so nothing else needs updating.
5. Style it in `styles.css` using existing tokens. Add a new section comment
   header if it is a genuinely new area.

## Adding a photograph

1. Drop the original into `assets/images/source/` named after the slot it fills.
2. Add an entry to `RECIPES` in `tools/build_images.py` with the widths that
   slot actually renders at, and a quality value.
3. Run `npm run images`.
4. Reference it with a `<picture>` carrying both the WebP and JPEG `srcset`, a
   `sizes` attribute matching the CSS, and `width`/`height` from
   `manifest.json` so nothing shifts while it loads.
